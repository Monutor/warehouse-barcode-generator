# Промт: PWA генератор штрих-кодов склада

> Для создания проекта с нуля — скопируй содержимое ниже в новый чат.

---

Создай PWA-приложение для генерации штрих-кодов полок склада. Приложение полностью клиентское (без бэкенда), хостится на GitHub Pages.

## Стек
- Vue 3 через CDN (без сборки, без npm)
- Bootstrap 5 через CDN (карточки, кнопки, сетка, утилиты)
- JsBarcode (CDN) для генерации Code-128
- manifest.json + service-worker.js для PWA (офлайн-режим)

## Данные
- Файл `data/shelves.js` — JS-массив с данными полок
- Формат: `{ name: "А-С01-1", barcode: "STL000004940001", section: "А", type: "С", number: "01", level: "1" }`
- Данные статичные, не меняются

## Конвертер CSV → JS
- Создай скрипт `scripts/convert_csv.py` — читает `warehouse_data.csv`, выводит `data/shelves.js`
- Запуск: `python scripts/convert_csv.py`
- Скрипт должен быть в репозитории, но результат конвертации (shelves.js) — тоже коммитится

## Бизнес-логика (портировать из Python warehouse.py)
1. Загрузка данных из shelves.js
2. Группировка по секциям: racks (стеллажи), pallets (поддоны)
3. Секция "СЛУЖЕБНАЯ" содержит zones (служебные зоны)
4. Секция "ПИКАП" содержит pallets
5. O(1) поиск по name через индекс
6. barcode из поля barcode, fallback — транслитерация
7. Статистика: total_sections, total_shelves, total_zones

## Транслитерация (портировать из Python translit.py)
- Маппинг кириллица → латиница для Code-128
- Только заглавные буквы

## Навигация (пошаговая, реактивная через v-if)
1. **Welcome** — приветствие, статистика, кнопка "Начать"
2. **Секции** — карточки секций, "Служебные зоны" в конце
3. **Тип хранения** — "Стеллажи" или "Поддоны"
4. **Стеллажи** — список → клик → уровни или штрих-код
5. **Уровни** — список → клик → штрих-код
6. **Поддоны** — список → клик → штрих-код
7. **Служебные зоны** — список → клик → штрих-код
8. **Штрих-код** — Code-128 + название полки
9. **ПИКАП** — список поддонов → клик → штрих-код

Каждый экран: кнопка "На главную" + "Назад" где уместно.

## Генерация штрих-кодов
- JsBarcode → SVG/Canvas Code-128
- Под штрих-кодом — название полки
- Кнопка "Скачать PNG"
- Кнопка "Ещё штрих-код" → Welcome
- Кэш в IndexedDB: при повторном просмотре того же штрих-кода — берётся из кэша, без перегенерации

## UI/UX
- Bootstrap 5: btn, card, container, row, col, d-flex, gap
- Мобильно-ориентированный дизайн (телефоны на складе)
- Крупные кнопки: btn-lg, py-3 (удобно в перчатках)
- Пагинация при > 12 элементов (Bootstrap pagination)
- Переходы через Vue <Transition>
- Только русский язык

## Обработка ошибок
- Если shelves.js не загрузился — показать дружелюбное сообщение: "Не удалось загрузить данные. Проверьте подключение и обновите страницу." + кнопка "Повторить"
- Если полка не найдена — "Штрих-код не найден для: <name>"
- Graceful degradation: если JsBarcode не загрузился — показать ошибку с предложением обновить

## PWA требования
- Service Worker кэширует все файлы (precache)
- Полностью офлайн после первого визита
- Установка на домашний экран
- manifest.json: name, short_name, icons, start_url, display: standalone

## Что НЕ нужно
- Без бэкенда, без API
- Без аутентификации
- Без админ-панели
- Без редактирования данных
- Без согласия деплоить проект на гитхаб
